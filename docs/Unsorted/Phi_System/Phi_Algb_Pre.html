text ‹Solves partial addition equations consisting of

▪ ‹given + ?unknown = given›, ‹?unknown + given = given›,
  ‹given = given + ?unknown›, ‹given = ?unknown + given›
▪ ‹given + ?unknown = ?unknown + given›, ‹?unknown + given = given + ?unknown› (only for non-commutative group)
▪ ‹?⇩+ ?unknown + ?⇩+ given + ?⇩+ ?unknown = ?⇩+ given›,
  ‹?⇩+ given = ?⇩+ ?unknown + ?⇩+ given + ?⇩+ ?unknown› (only for non-commutative group)

NOTE, the reasoning ignores any case when the given in the LHS equals the given in the RHS.
You MUST check the trivial case before calling the reasoning process if can happen.

Note some forms are only meaningful for non-commutative group as if not they can be reduced to the
first form.
Also not, as addition can be associative, we use ‹id› to annotate explicitly each element in the equation.
For instance, ‹id a + id b + id c = id d› to distinguish with ‹id (a + b) + id c = id d›.

System rules first normalize the problem into one of
▪ ‹equation⇩2⇩1› ‹?unknown_d + given_a = given_b› (the only property required in commutative algebra) or
  ‹equation⇩2⇩1› ‹given_b + ?unknown_c = given_a› (only for non-commutative algebra)
▪ ‹dabc_equation ?unknown given given ?unknown› (only for non-commutative algebra)
▪ ‹equation⇩3⇩1› ‹?⇩+ ?unknown + ?⇩+ given + ?⇩+ ?unknown = ?⇩+ given› (necessary for non-commutative algebra, optional
    for commutative algebra in which it reduces to ‹?unknown_d + given_a = given_b›)

Then the above three forms are what user rules have to handle for specific algebras.

There are pre-built reasoning rules for,
▪ cancellative and canonically ordered commutative monoid, including the version for both partial algebras
  and total. This is the weakest algebraic structure to have a general algorithm,
  if an algorithm deciding the order relation is assumed.
▪ group, (only that for total algebra is installed), which is though not the minimal requirement,
  the weakest one available in Isabelle standard library, as the minimal one, quasigroup, is not
  formalized in Isabelle standard library.
›
