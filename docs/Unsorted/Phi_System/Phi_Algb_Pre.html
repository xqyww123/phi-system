text ‚ÄπSolves partial addition equations consisting of

‚ñ™ ‚Äπgiven + ?unknown = given‚Ä∫, ‚Äπ?unknown + given = given‚Ä∫,
  ‚Äπgiven = given + ?unknown‚Ä∫, ‚Äπgiven = ?unknown + given‚Ä∫
‚ñ™ ‚Äπgiven + ?unknown = ?unknown + given‚Ä∫, ‚Äπ?unknown + given = given + ?unknown‚Ä∫ (only for non-commutative group)
‚ñ™ ‚Äπ?‚á©+ ?unknown + ?‚á©+ given + ?‚á©+ ?unknown = ?‚á©+ given‚Ä∫,
  ‚Äπ?‚á©+ given = ?‚á©+ ?unknown + ?‚á©+ given + ?‚á©+ ?unknown‚Ä∫ (only for non-commutative group)

NOTE, the reasoning ignores any case when the given in the LHS equals the given in the RHS.
You MUST check the trivial case before calling the reasoning process if can happen.

Note some forms are only meaningful for non-commutative group as if not they can be reduced to the
first form.
Also not, as addition can be associative, we use ‚Äπid‚Ä∫ to annotate explicitly each element in the equation.
For instance, ‚Äπid a + id b + id c = id d‚Ä∫ to distinguish with ‚Äπid (a + b) + id c = id d‚Ä∫.

System rules first normalize the problem into one of
‚ñ™ ‚Äπequation‚á©2‚á©1‚Ä∫ ‚Äπ?unknown_d + given_a = given_b‚Ä∫ (the only property required in commutative algebra) or
  ‚Äπequation‚á©2‚á©1‚Ä∫ ‚Äπgiven_b + ?unknown_c = given_a‚Ä∫ (only for non-commutative algebra)
‚ñ™ ‚Äπdabc_equation ?unknown given given ?unknown‚Ä∫ (only for non-commutative algebra)
‚ñ™ ‚Äπequation‚á©3‚á©1‚Ä∫ ‚Äπ?‚á©+ ?unknown + ?‚á©+ given + ?‚á©+ ?unknown = ?‚á©+ given‚Ä∫ (necessary for non-commutative algebra, optional
    for commutative algebra in which it reduces to ‚Äπ?unknown_d + given_a = given_b‚Ä∫)

Then the above three forms are what user rules have to handle for specific algebras.

There are pre-built reasoning rules for,
‚ñ™ cancellative and canonically ordered commutative monoid, including the version for both partial algebras
  and total. This is the weakest algebraic structure to have a general algorithm,
  if an algorithm deciding the order relation is assumed.
‚ñ™ group, (only that for total algebra is installed), which is though not the minimal requirement,
  the weakest one available in Isabelle standard library, as the minimal one, quasigroup, is not
  formalized in Isabelle standard library.
‚Ä∫
